跨度类
=====

`runtime.spanClass`是`runtime.mspan`的跨度类，它决定了内存管理单元中存储的的对象大小和数量
```go
type mspan struct {
	...
	spanclass   spanClass
	...
}
```
Go语言的内存管理模块中一共包含了67个跨度类，每1个跨度类都会存储特定大小的对象，并且包含固定的页数和对象数
所有的数据都会被预先计算好放在 `runtime.class_to_size` 和`class_to_allocnpages`等变量中
可以参考文件 `runtime/sizeclasses.go`

**跨度类的数据**

上表展示了67种跨度类，存储对象的大小从8B到32KB,另外还存储了总内存占用,对象数和浪费的空间。
我们来具体看下class=66这个跨度类：
`mspan(class=66)`
```go
//    66      28672       57344        2           0      4.91%       4096
```

这个跨度类存储的对象大小的范围是 [27265,28672],包含页数为 57344/8K = 7,最多可以存储对象数量为2，
因为内存需要按照页来进行管理，尾部需要浪费的空间是 0 ,当页中存储的对象都是27265大小时，
总共浪费的空间是 (28672-27265)*2 = 2814 ,2814占总内存57344的`4.91%`

除了上述67个跨度类之外，运行时中还包含了ID为0的特殊跨度类，它能够管理大于32KB的特殊对象。

跨度类中除了存储了ID外，它还会存储`noscan`标记位,该标记位表示对象是否包含指针，
垃圾回收会对包含指针的 `runtime.mspan` 进行扫描，通过以下代码观察到，spanclass
```go
func makeSpanClass(sizeclass uint8, noscan bool) spanClass {
	return spanClass(sizeclass<<1) | spanClass(bool2int(noscan))
}

func (sc spanClass) sizeclass() int8 {
	return int8(sc >> 1)
}

func (sc spanClass) noscan() bool {
	return sc&1 != 0
}
```

runtime.spanClass 是一个 uint8 类型的整数，它的前 7 位存储着跨度类的 ID，最后一位表示是否包含指针，该类型提供的两个方法能够帮我们快速获取对应的字段。
